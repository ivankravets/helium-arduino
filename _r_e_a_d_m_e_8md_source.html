12 # Getting Started #
13 
14 
15 ## Hardware ##
16 
17 To get started with %Helium for Arduino you will need the following hardware:
18 
19 * A %Helium Atom. The easiest way to get started is to get
20  the
21  [Helium Starter Kit](https://www.helium.com/products/helium-starter-kit) which
22  includes a gateway (Element), and an XBee version of the %Helium
23  Atom.
24 
25 * An Arduino board of any supported type, such as
26  the
27  [Arduino Uno](https://www.arduino.cc/en/main/arduinoBoardUno). The
28  supported boards are listed below.
29 
30 * An
31  [Arduino XBee Shield](https://www.helium.com/products/atom-arduino-adapters) to
32  adapt the XBee to the Arduino shield pinout.
33

36 ## Software ##
37 
38 You will need the following software installed:
39 
40 * The [Arduino IDE](https://www.arduino.cc/en/Main/Software).
41 
42 * The
43  [Helium Arduino Library](https://github.com/helium/helium-arduino). You
44  can install this library through either of the following steps:
45 
46  * The Library Manager by clicking on _Sketch > Include Library_,
47  searching for %Helium and clicking the _Install_ button.
48 
49  * By downloading a release _Helium.zip_ file from
50  the
51  [helium-arduino releases](https://github.com/helium/helium-arduino/releases) page. Once
52  downloaded launch the Arduino IDE, click _Sketch > Include
53  Library, Add .Zip Library_, and selecting the _Helium.zip_
54  file. Note that you will need to re-start the Arduino IDE after
55  installing the zip library.
56

58 ## Board Setup ##
59 
60 Ensure that the %Helium Element is powered on and
61 connected. The main LED on the Element should be green or blue to
62 indicate it is powered on and connected to the network.
63 
64 Plug the Adapter into the Arduino board and the %Helium XBee Atom into
65 the Adapter.
66 
67 Depending on the Arduino board you will have to configure the
68 D<SUB>in</SUB> and D<SUB>out</SUB> jumpers on the Adapter to get the
69 Arduino to talk to the %Helium Atom using the right pins.
70 
71 The following boards D<SUB>in</SUB> and D<SUB>out</SUB> pins are
72 known. Other setups will also work but we've confirmed the following
73 work.
74 
75 Board | D<SUB>in</SUB> | D<SUB>out</SUB>
76 ------------------| -------------: | --------------:
77 Arduino Uno | 8 | 9
78 Arduino M0 Pro | 0 | 1
79 Arduino Zero | 0 | 1
80 
81 ## Example Setup ##
82 
83 The example sketches can be found in the Arduino IDE under _File > Examples > Helium_.
84 
85 With one of the examples opened, the Element powered on, and the
86 configured board connected to a USB port click the _Run_ button.
87 
88 The IDE should upload the example sketch to the board and you should
89 see the Atom power on, connect and perform the task coded into the
90 loaded example sketch.
91 
92 Note that some of the demo sketches will require you to have a
93 configured %Channel in the %Helium Dashboard in order to channel data
94 from the device all the way to the end IoT service.
95 
96 Please read the comments in the example sketch and in the %Channel
97 configuration sections in the dashboard for more information on how to
98 set up and configure channels.
